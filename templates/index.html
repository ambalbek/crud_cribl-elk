Capabilities

📝

Submit Onboarding Requests

Clients fill a form with App ID, App Name, Region, and Entitlement Groups. Requests are indexed in Elasticsearch with a unique Request ID.

🚀

Push Cribl Routes

Inject new Cribl routes above the default catch-all, targeting a specific pipeline and destination, across one or more worker groups.

⚡

ELK Roles + Cribl Routes

Create ELK security roles and role-mappings in nonprod and prod, and inject Cribl routes and destinations — in a single coordinated operation.

📋

Single or Bulk

Process one app by ID + name, or upload a .txt file with multiple apps (appid, appname per line).

🔒

Safe by Default

Dry-run is checked by default. Protected workspaces require an explicit "Allow production writes" override before any changes are made.

🔄

End-to-End Lifecycle

From request submission through processing to status update — the entire onboarding lifecycle is managed in one place.

🔒

Entitlement Lookup

View entitlement-to-role mappings across all ELK clusters. Search by cluster, entitlement CN, or role name. Export results to CSV.

Workflow

  1. 1

    Authenticate

    User visits the app, is redirected to /login, and enters username + password. Role (user or admin) is assigned based on account type.

  2. 2

    Client submits an onboarding request

    Open the Onboarding Portal — username and name are auto-populated from the SSO session. Fill in App ID, App Name, Region, and Entitlement Groups.

  3. 3

    Platform team reviews and processes

    Open the Cribl Pusher (admin only), select workspace and worker group(s), enter the app details, and run with Dry Run first.

  4. 4

    Routes and destinations are created

    Uncheck Dry Run and re-run. Routes are inserted above the catch-all, destinations are created, and ELK roles are provisioned.

  5. 5

    Request status is updated

    After successful processing, use the Admin panel (admin only) to mark the request as "done".

  6. 6

    Verify entitlements

    Open the Entitlement Lookup page to browse role mappings across all ELK clusters. Search, filter, and export to CSV.

Entitlement Lookup Flow

Open /entitlements ➔ Query ES clusters ➔ Extract role mappings ➔ Filter by entitlement ➔ Searchable table + CSV

Credential Resolution Order

Credentials are resolved in this order: UI field → environment variable → config.json. Leave UI fields blank to fall back to config.json. 
# Environment variable fallbacks
export CRIBL_TOKEN=your-token
export CRIBL_USERNAME=admin
export CRIBL_PASSWORD=secret

Safety Notes

Dry Run is on by default. Always review the diff output before unchecking it. Routes are inserted above the first final: true route — verify the position is correct in dry-run output.
Protected workspaces require the Allow production writes checkbox to be checked. The run will be rejected otherwise.
